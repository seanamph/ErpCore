# 故障排除文件

## 概述

本文檔提供 ErpCore 系統常見問題的解決方案。

## 常見問題

### 資料庫連線問題

**問題**: 無法連線到資料庫

**解決方案**:
1. 檢查 `appsettings.json` 中的連線字串
2. 確認 SQL Server 服務正在運行
3. 檢查防火牆設定
4. 確認資料庫使用者權限

### API 無法啟動

**問題**: API 啟動失敗

**解決方案**:
1. 檢查日誌檔案（NLog 日誌）
2. 確認所有依賴套件已安裝
3. 檢查連接埠是否被占用
4. 確認環境變數設定正確

### 前端無法連線 API

**問題**: 前端無法呼叫後端 API

**解決方案**:
1. 檢查 API 端點設定（`.env` 檔案）
2. 確認 CORS 設定正確
3. 檢查瀏覽器控制台錯誤訊息
4. 確認 API 服務正在運行

### 日誌檔案問題

**問題**: 日誌檔案未產生

**解決方案**:
1. 檢查 `nlog.config` 設定
2. 確認日誌目錄權限
3. 檢查 NLog 套件版本
4. 確認日誌級別設定

### 建置失敗

**問題**: 專案建置失敗

**解決方案**:
1. 清理專案：`dotnet clean`
2. 還原套件：`dotnet restore`
3. 檢查專案檔案的套件引用
4. 確認 .NET SDK 版本正確

## 日誌位置

- **NLog 日誌**: `logs/` 目錄（按小時分割，檔名格式：yyyyMMdd-HH.log）
- **應用程式日誌**: 根據 `appsettings.json` 設定
