# 考勤報表查詢 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: AttendanceReport
- **功能名稱**: 考勤報表查詢
- **功能描述**: 提供考勤相關報表的查詢、統計、匯出功能，包含出勤統計、請假統計、加班統計、異常統計等報表
- **參考舊程式**: 
  - `WEB/IMS_CORE/ASP/Attendance/` 報表相關檔案
  - `IMS3/HANSHIN/IMS3/Attendance/` 報表相關檔案

### 1.2 業務需求
- 提供員工出勤統計報表
- 提供請假統計報表
- 提供加班統計報表
- 提供考勤異常統計報表
- 支援多條件查詢與篩選
- 支援報表匯出 (Excel/PDF)
- 支援報表列印

## 二、資料庫設計 (Schema)

### 2.1 主要資料表

本功能主要查詢以下資料表：
- `Attendance` - 考勤記錄表（參考考勤資料維護）
- `LeaveApplications` - 請假申請表（參考考勤資料維護）
- `OvertimeApplications` - 加班申請表（參考考勤資料維護）
- `Employees` - 員工主檔

### 2.2 報表統計視圖

#### 2.2.1 `v_AttendanceSummary` - 考勤統計視圖
```sql
CREATE VIEW [dbo].[v_AttendanceSummary] AS
SELECT 
    a.EmployeeId,
    e.EmployeeName,
    e.DepartmentId,
    d.DepartmentName,
    YEAR(a.AttendanceDate) AS Year,
    MONTH(a.AttendanceDate) AS Month,
    COUNT(*) AS TotalDays,
    SUM(CASE WHEN a.Status = 'NORMAL' THEN 1 ELSE 0 END) AS NormalDays,
    SUM(CASE WHEN a.Status = 'LATE' THEN 1 ELSE 0 END) AS LateDays,
    SUM(CASE WHEN a.Status = 'EARLY' THEN 1 ELSE 0 END) AS EarlyDays,
    SUM(CASE WHEN a.Status = 'ABSENT' THEN 1 ELSE 0 END) AS AbsentDays,
    SUM(CASE WHEN a.Status = 'LEAVE' THEN 1 ELSE 0 END) AS LeaveDays,
    SUM(a.WorkHours) AS TotalWorkHours,
    SUM(a.OvertimeHours) AS TotalOvertimeHours,
    SUM(a.LeaveHours) AS TotalLeaveHours
FROM Attendance a
INNER JOIN Employees e ON a.EmployeeId = e.EmployeeId
LEFT JOIN Departments d ON e.DepartmentId = d.DepartmentId
GROUP BY a.EmployeeId, e.EmployeeName, e.DepartmentId, d.DepartmentName, 
         YEAR(a.AttendanceDate), MONTH(a.AttendanceDate);
```

#### 2.2.2 `v_LeaveSummary` - 請假統計視圖
```sql
CREATE VIEW [dbo].[v_LeaveSummary] AS
SELECT 
    la.EmployeeId,
    e.EmployeeName,
    e.DepartmentId,
    d.DepartmentName,
    YEAR(la.StartDate) AS Year,
    MONTH(la.StartDate) AS Month,
    la.LeaveType,
    COUNT(*) AS LeaveCount,
    SUM(la.TotalHours) AS TotalLeaveHours,
    SUM(CASE WHEN la.Status = 'APPROVED' THEN la.TotalHours ELSE 0 END) AS ApprovedLeaveHours
FROM LeaveApplications la
INNER JOIN Employees e ON la.EmployeeId = e.EmployeeId
LEFT JOIN Departments d ON e.DepartmentId = d.DepartmentId
GROUP BY la.EmployeeId, e.EmployeeName, e.DepartmentId, d.DepartmentName,
         YEAR(la.StartDate), MONTH(la.StartDate), la.LeaveType;
```

#### 2.2.3 `v_OvertimeSummary` - 加班統計視圖
```sql
CREATE VIEW [dbo].[v_OvertimeSummary] AS
SELECT 
    oa.EmployeeId,
    e.EmployeeName,
    e.DepartmentId,
    d.DepartmentName,
    YEAR(oa.OvertimeDate) AS Year,
    MONTH(oa.OvertimeDate) AS Month,
    COUNT(*) AS OvertimeCount,
    SUM(oa.OvertimeHours) AS TotalOvertimeHours,
    SUM(CASE WHEN oa.Status = 'APPROVED' THEN oa.OvertimeHours ELSE 0 END) AS ApprovedOvertimeHours
FROM OvertimeApplications oa
INNER JOIN Employees e ON oa.EmployeeId = e.EmployeeId
LEFT JOIN Departments d ON e.DepartmentId = d.DepartmentId
GROUP BY oa.EmployeeId, e.EmployeeName, e.DepartmentId, d.DepartmentName,
         YEAR(oa.OvertimeDate), MONTH(oa.OvertimeDate);
```

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢考勤統計報表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/attendance/reports/summary`
- **說明**: 查詢考勤統計報表，支援多條件篩選
- **請求參數**:
  ```json
  {
    "employeeId": "",
    "departmentId": "",
    "startDate": "2024-01-01",
    "endDate": "2024-01-31",
    "groupBy": "MONTH", // DAY, WEEK, MONTH, YEAR
    "pageIndex": 1,
    "pageSize": 20
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "items": [
        {
          "employeeId": "E001",
          "employeeName": "張三",
          "departmentName": "資訊部",
          "year": 2024,
          "month": 1,
          "totalDays": 22,
          "normalDays": 20,
          "lateDays": 1,
          "earlyDays": 0,
          "absentDays": 0,
          "leaveDays": 1,

  }
  ```

#### 3.1.5 匯出報表 (Excel)
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/attendance/reports/export/excel`
- **說明**: 匯出考勤報表為 Excel 格式
- **請求格式**:
  ```json
  {
    "reportType": "SUMMARY", // SUMMARY, LEAVE, OVERTIME, ABNORMAL
    "filters": {
      "employeeId": "",
      "departmentId": "",
      "startDate": "2024-01-01",
      "endDate": "2024-01-31"
    }
  }
  ```
- **回應格式**: 返回 Excel 檔案下載連結或直接下載

#### 3.1.6 匯出報表 (PDF)
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/attendance/reports/export/pdf`
- **說明**: 匯出考勤報表為 PDF 格式
- **請求格式**: 同 Excel 匯出
- **回應格式**: 返回 PDF 檔案下載連結或直接下載

## 六、注意事項

### 6.1 安全性
- 敏感資料必須加密傳輸 (HTTPS)
- 必須實作權限檢查
- 報表資料需要權限控制

### 6.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 統計查詢需要使用視圖優化
- 匯出大檔案需要非同步處理

### 6.3 資料驗證
- 日期範圍必須在有效範圍內
- 分組方式必須在允許範圍內
- 報表類型必須在允許範圍內

### 6.4 業務邏輯
- 報表資料需要即時計算
- 匯出檔案需要限制大小
- 列印需要格式化處理
