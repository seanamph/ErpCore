# SYS6310-SYS6370 - 樓層資料維護系列 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: SYS6310-SYS6370 系列
- **功能名稱**: 樓層資料維護系列
- **功能描述**: 提供樓層基本資料的新增、修改、刪除、查詢功能，包含樓層編號、樓層名稱、樓層號碼、樓層描述、狀態等資訊管理
- **參考舊程式**: 
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6310_FB.ASP` (瀏覽)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6310_FI.ASP` (新增)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6310_FU.ASP` (修改)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6310_FD.ASP` (刪除)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6310_FQ.ASP` (查詢)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6310_PR.ASP` (報表)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6320_*.ASP` (樓層相關功能)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6330_*.ASP` (樓層相關功能)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6340_*.ASP` (樓層相關功能)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6350_*.ASP` (樓層相關功能)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6360_*.ASP` (樓層相關功能)
  - `WEB/IMS_CORE/ASP/SYS6000/SYS6370_*.ASP` (樓層相關功能)

### 1.2 業務需求
- 管理樓層基本資料資訊
- 支援樓層的新增、修改、刪除、查詢
- 記錄樓層的建立與變更資訊
- 支援樓層號碼管理
- 支援樓層狀態管理（啟用/停用）
- 支援樓層商店關聯查詢

## 二、資料庫設計 (Schema)

### 2.1 主要資料表: `Floors` (樓層主檔)

```sql
CREATE TABLE [dbo].[Floors] (
    [FloorId] NVARCHAR(50) NOT NULL PRIMARY KEY,
    [FloorName] NVARCHAR(100) NOT NULL,
    [FloorNameEn] NVARCHAR(100) NULL,
    [FloorNumber] INT NULL, -- 樓層號碼
    [Description] NVARCHAR(500) NULL,
    [Status] NVARCHAR(10) NOT NULL DEFAULT 'A', -- A:啟用, I:停用
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [UpdatedBy] NVARCHAR(50) NULL,
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [PK_Floors] PRIMARY KEY CLUSTERED ([FloorId] ASC)
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_Floors_FloorNumber] ON [dbo].[Floors] ([FloorNumber]);
CREATE NONCLUSTERED INDEX [IX_Floors_Status] ON [dbo].[Floors] ([Status]);
```

### 2.2 資料字典

| 欄位名稱 | 資料類型 | 長度 | 允許NULL | 預設值 | 說明 | 備註 |

| FloorId | NVARCHAR | 50 | NO | - | 樓層編號 | 主鍵，唯一 |
| FloorName | NVARCHAR | 100 | NO | - | 樓層名稱 | - |
| FloorNameEn | NVARCHAR | 100 | YES | - | 樓層英文名稱 | - |
| FloorNumber | INT | - | YES | - | 樓層號碼 | - |
| Description | NVARCHAR | 500 | YES | - | 樓層描述 | - |

| CreatedBy | NVARCHAR | 50 | YES | - | 建立者 | - |

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢樓層列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/floors`
- **說明**: 查詢樓層列表，支援分頁、排序、篩選
- **請求參數**:
  ```json
  {
    "pageIndex": 1,
    "pageSize": 20,
    "sortField": "FloorNumber",
    "sortOrder": "ASC",
    "filters": {
      "floorId": "",
      "floorName": "",
      "status": ""
    }
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "items": [
        {
          "floorId": "FLOOR001",
          "floorName": "一樓",
          "floorNameEn": "1F",
          "floorNumber": 1,
          "description": "一樓描述",
          "status": "A"
        }
      ],
      "totalCount": 10,
      "pageIndex": 1,
      "pageSize": 20,
      "totalPages": 1
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.2 查詢單筆樓層
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/floors/{floorId}`
- **說明**: 根據樓層編號查詢單筆樓層資料
- **路徑參數**:
  - `floorId`: 樓層編號
- **回應格式**: 同查詢列表單筆資料格式

#### 3.1.3 新增樓層
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/floors`
- **說明**: 新增樓層資料
- **請求格式**:
  ```json
  {
    "floorId": "FLOOR001",
    "floorName": "一樓",
    "floorNameEn": "1F",
    "floorNumber": 1,
    "description": "一樓描述",
    "status": "A"
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "新增成功",
    "data": {
      "floorId": "FLOOR001"
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.4 修改樓層
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/floors/{floorId}`
- **說明**: 修改樓層資料
- **路徑參數**:
  - `floorId`: 樓層編號
- **請求格式**: 同新增，但 `floorId` 不可修改
- **回應格式**: 同新增

#### 3.1.5 刪除樓層
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/floors/{floorId}`
- **說明**: 刪除樓層資料（軟刪除或硬刪除）
- **路徑參數**:
  - `floorId`: 樓層編號
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "刪除成功",
    "data": null,
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.6 啟用/停用樓層
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/floors/{floorId}/status`
- **說明**: 啟用或停用樓層
- **請求格式**:
  ```json
  {
    "status": "A" // A:啟用, I:停用
  }
  ```

### 4.3 API 呼叫 (`floor.api.ts`)
```typescript
import request from '@/utils/request';

export interface FloorDto {
  floorId: string;
  floorName: string;
  floorNameEn?: string;
  floorNumber?: number;
  description?: string;
  status: string;
}

export interface FloorQueryDto {
  pageIndex: number;
  pageSize: number;
  sortField?: string;
  sortOrder?: 'ASC' | 'DESC';
  filters?: {
    floorId?: string;
    floorName?: string;
    status?: string;
  };
}

export interface CreateFloorDto {
  floorId: string;
  floorName: string;
  floorNameEn?: string;
  floorNumber?: number;
  description?: string;
  status: string;
}

export interface UpdateFloorDto extends Omit<CreateFloorDto, 'floorId'> {}

// API 函數
export const getFloorList = (query: FloorQueryDto) => {
  return request.get<ApiResponse<PagedResult<FloorDto>>>('/api/v1/floors', { params: query });
};

export const getFloorById = (floorId: string) => {
  return request.get<ApiResponse<FloorDto>>(`/api/v1/floors/${floorId}`);
};

export const createFloor = (data: CreateFloorDto) => {
  return request.post<ApiResponse<string>>('/api/v1/floors', data);
};

export const updateFloor = (floorId: string, data: UpdateFloorDto) => {
  return request.put<ApiResponse>(`/api/v1/floors/${floorId}`, data);
};

export const deleteFloor = (floorId: string) => {
  return request.delete<ApiResponse>(`/api/v1/floors/${floorId}`);
};

export const updateFloorStatus = (floorId: string, status: string) => {
  return request.put<ApiResponse>(`/api/v1/floors/${floorId}/status`, { status });
};
```

## 六、注意事項

### 6.1 安全性
- 必須實作權限檢查
- 敏感資料必須加密傳輸 (HTTPS)
- 必須驗證使用者輸入

### 6.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 必須使用快取機制

### 6.3 資料驗證
- 樓層編號必須唯一
- 必填欄位必須驗證
- 狀態值必須在允許範圍內
- 樓層號碼必須為正整數

### 6.4 業務邏輯
- 刪除樓層前必須檢查是否有相關商店
- 停用樓層時必須檢查是否有進行中的業務
- 樓層號碼建議唯一
