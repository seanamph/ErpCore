# SYS3130-SYS3160 - 商店資料維護系列 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: SYS3130-SYS3160 系列
- **功能名稱**: 商店資料維護系列
- **功能描述**: 提供商店基本資料的新增、修改、刪除、查詢功能，包含商店編號、商店名稱、地址、聯絡資訊、POS設定、商店類型等資訊管理
- **參考舊程式**: 
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3130_FB.ASP` (瀏覽)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3130_FI.ASP` (新增)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3130_FU.ASP` (修改)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3130_FD.ASP` (刪除)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3130_FQ.ASP` (查詢)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3130_PR.ASP` (報表)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3135_*.ASP` (商店相關功能)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3140_*.ASP` (商店相關功能)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3145_*.ASP` (商店相關功能)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3150_*.ASP` (商店相關功能)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3160_*.ASP` (商店相關功能)

### 1.2 業務需求
- 管理商店基本資料資訊
- 支援商店的新增、修改、刪除、查詢
- 記錄商店的建立與變更資訊
- 支援商店類型設定
- 支援POS系統整合
- 支援商店地址與聯絡資訊管理
- 支援商店狀態管理（啟用/停用）

## 二、資料庫設計 (Schema)

### 2.1 主要資料表: `Shops`

```sql
CREATE TABLE [dbo].[Shops] (
    [ShopId] NVARCHAR(50) NOT NULL PRIMARY KEY,
    [ShopName] NVARCHAR(200) NOT NULL,
    [ShopNameEn] NVARCHAR(200) NULL,
    [ShopType] NVARCHAR(50) NULL,
    [Address] NVARCHAR(500) NULL,
    [City] NVARCHAR(50) NULL,
    [Zone] NVARCHAR(50) NULL,
    [PostalCode] NVARCHAR(20) NULL,
    [Phone] NVARCHAR(50) NULL,
    [Fax] NVARCHAR(50) NULL,
    [Email] NVARCHAR(100) NULL,
    [ManagerName] NVARCHAR(100) NULL,
    [ManagerPhone] NVARCHAR(50) NULL,
    [OpenDate] DATETIME2 NULL,
    [CloseDate] DATETIME2 NULL,
    [Status] NVARCHAR(10) NOT NULL DEFAULT 'A', -- A:啟用, I:停用
    [FloorId] NVARCHAR(50) NULL,
    [AreaId] NVARCHAR(50) NULL,
    [PosEnabled] BIT NULL DEFAULT 0,
    [PosSystemId] NVARCHAR(50) NULL,
    [Notes] NVARCHAR(1000) NULL,
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [UpdatedBy] NVARCHAR(50) NULL,
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [PK_Shops] PRIMARY KEY CLUSTERED ([ShopId] ASC)
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_Shops_ShopType] ON [dbo].[Shops] ([ShopType]);
CREATE NONCLUSTERED INDEX [IX_Shops_Status] ON [dbo].[Shops] ([Status]);
CREATE NONCLUSTERED INDEX [IX_Shops_FloorId] ON [dbo].[Shops] ([FloorId]);
CREATE NONCLUSTERED INDEX [IX_Shops_AreaId] ON [dbo].[Shops] ([AreaId]);
CREATE NONCLUSTERED INDEX [IX_Shops_City] ON [dbo].[Shops] ([City]);
```

### 2.2 相關資料表

#### 2.2.1 `ShopTypes` - 商店類型
```sql
CREATE TABLE [dbo].[ShopTypes] (
    [ShopTypeId] NVARCHAR(50) NOT NULL PRIMARY KEY,
    [ShopTypeName] NVARCHAR(100) NOT NULL,

    [Description] NVARCHAR(500) NULL,
    [Status] NVARCHAR(10) NOT NULL DEFAULT 'A',
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [UpdatedBy] NVARCHAR(50) NULL,
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE()
);
```

#### 2.2.2 `ShopPosSettings` - 商店POS設定
```sql
CREATE TABLE [dbo].[ShopPosSettings] (
    [ShopId] NVARCHAR(50) NOT NULL,
    [PosSystemId] NVARCHAR(50) NOT NULL,
    [PosTerminalId] NVARCHAR(50) NULL,
    [PosConfig] NVARCHAR(MAX) NULL, -- JSON格式的POS設定
    [Status] NVARCHAR(10) NOT NULL DEFAULT 'A',
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [UpdatedBy] NVARCHAR(50) NULL,
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [PK_ShopPosSettings] PRIMARY KEY CLUSTERED ([ShopId], [PosSystemId]),
    CONSTRAINT [FK_ShopPosSettings_Shops] FOREIGN KEY ([ShopId]) REFERENCES [dbo].[Shops] ([ShopId]) ON DELETE CASCADE
);
```

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢商店列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/shops`
- **說明**: 查詢商店列表，支援分頁、排序、篩選
- **請求參數**:
  ```json
  {
    "pageIndex": 1,
    "pageSize": 20,
    "sortField": "ShopId",
    "sortOrder": "ASC",
    "filters": {
      "shopId": "",
      "shopName": "",
      "shopType": "",
      "status": "",
      "city": "",
      "floorId": "",
      "areaId": ""
    }
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "items": [
        {
          "shopId": "SHOP001",
          "shopName": "測試商店",
          "shopNameEn": "Test Shop",
          "shopType": "TYPE001",
          "address": "測試地址",
          "city": "台北市",
          "zone": "信義區",
          "phone": "02-12345678",
          "status": "A",
          "openDate": "2024-01-01",
          "posEnabled": true
        }
      ],
      "totalCount": 100,
      "pageIndex": 1,
      "pageSize": 20,
      "totalPages": 5
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.2 查詢單筆商店
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/shops/{shopId}`
- **說明**: 根據商店編號查詢單筆商店資料
- **路徑參數**:
  - `shopId`: 商店編號
- **回應格式**: 同查詢列表單筆資料格式

#### 3.1.3 新增商店
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/shops`
- **說明**: 新增商店資料
- **請求格式**:
  ```json
  {
    "shopId": "SHOP001",
    "shopName": "測試商店",
    "shopNameEn": "Test Shop",
    "shopType": "TYPE001",
    "address": "測試地址",
    "city": "台北市",
    "zone": "信義區",
    "postalCode": "110",
    "phone": "02-12345678",
    "fax": "02-12345679",
    "email": "test@example.com",
    "managerName": "張三",
    "managerPhone": "0912345678",
    "openDate": "2024-01-01",
    "closeDate": null,
    "status": "A",
    "floorId": "FLOOR001",
    "areaId": "AREA001",
    "posEnabled": true,
    "posSystemId": "POS001",
    "notes": "備註"
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "新增成功",
    "data": {
      "shopId": "SHOP001"
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.4 修改商店
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/shops/{shopId}`
- **說明**: 修改商店資料
- **路徑參數**:
  - `shopId`: 商店編號
- **請求格式**: 同新增，但 `shopId` 不可修改
- **回應格式**: 同新增

#### 3.1.5 刪除商店
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/shops/{shopId}`
- **說明**: 刪除商店資料（軟刪除或硬刪除）
- **路徑參數**:
  - `shopId`: 商店編號
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "刪除成功",
    "data": null,
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.6 批次刪除商店
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/shops/batch`
- **說明**: 批次刪除多筆商店
- **請求格式**:
  ```json
  {
    "shopIds": ["SHOP001", "SHOP002", "SHOP003"]
  }
  ```

#### 3.1.7 啟用/停用商店
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/shops/{shopId}/status`
- **說明**: 啟用或停用商店
- **請求格式**:
  ```json
  {
    "status": "A" // A:啟用, I:停用
  }
  ```

### 4.3 API 呼叫 (`shop.api.ts`)
```typescript
import request from '@/utils/request';

export interface ShopDto {
  shopId: string;
  shopName: string;
  shopNameEn?: string;
```

## 六、注意事項

### 6.1 安全性
- 必須實作權限檢查
- 敏感資料必須加密傳輸 (HTTPS)
- 必須驗證使用者輸入

### 6.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 必須使用快取機制

### 6.3 資料驗證
- 商店編號必須唯一
- 必填欄位必須驗證
- 日期範圍必須驗證
- 狀態值必須在允許範圍內
- 電子郵件格式必須驗證

### 6.4 業務邏輯
- 刪除商店前必須檢查是否有相關資料（會員、POS交易等）
- 停用商店時必須檢查是否有進行中的業務
- POS設定必須與POS系統整合驗證
