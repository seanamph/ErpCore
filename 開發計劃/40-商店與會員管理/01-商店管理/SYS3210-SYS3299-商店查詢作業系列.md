### 1.1 功能說明

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢商店列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/shops/query`
- **說明**: 查詢商店列表，支援多種查詢條件、排序、分頁
- **請求參數**:
  ```json
  {
    "pageIndex": 1,
    "pageSize": 20,
    "sortField": "ShopId",
    "sortOrder": "ASC",
    "filters": {
      "shopId": "",
      "shopName": "",
      "shopType": "",
      "status": "",
      "city": "",
      "zone": "",
      "managerName": "",
      "startDate": "",
      "endDate": ""
    }
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "items": [
        {
          "shopId": "SHOP001",
          "shopName": "台北店",
          "shopType": "直營店",
          "status": "A",
          "city": "台北市",
          "zone": "信義區",
          "managerName": "張三",
          "phone": "02-12345678",
          "createdAt": "2024-01-01T00:00:00"
        }
      ],
      "totalCount": 100,
      "pageIndex": 1,
      "pageSize": 20,
      "totalPages": 5
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.2 匯出查詢結果
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/shops/query/export`
- **說明**: 匯出查詢結果為 Excel 或 PDF
- **請求格式**:
  ```json
  {
    "exportType": "Excel", // Excel 或 PDF
    "filters": {
      "shopId": "",
      "shopName": "",
      "shopType": "",
      "status": "",
      "city": "",
      "zone": ""
    },
    "columns": ["shopId", "shopName", "shopType", "status", "city", "zone"]
  }
  ```
- **回應格式**: 返回檔案下載連結或直接下載

### 4.3 API 呼叫 (`shopQuery.api.ts`)
```typescript
import request from '@/utils/request';

export interface ShopQueryDto {
  shopId: string;
  shopName: string;
  shopType?: string;
  status: string;
  city?: string;
  zone?: string;
  managerName?: string;
  phone?: string;
  createdAt: string;
}

export interface ShopQueryRequestDto {
  pageIndex: number;
  pageSize: number;
  sortField?: string;
  sortOrder?: 'ASC' | 'DESC';
  filters?: {
    shopId?: string;
    shopName?: string;
    shopType?: string;
    status?: string;
    city?: string;
    zone?: string;
    managerName?: string;
    startDate?: string;
    endDate?: string;
  };
}

export interface ShopExportRequestDto {
  exportType: 'Excel' | 'PDF';
  filters?: {
    shopId?: string;
    shopName?: string;
    shopType?: string;
    status?: string;
    city?: string;
    zone?: string;
  };
  columns?: string[];
}

// API 函數
export const queryShops = (request: ShopQueryRequestDto) => {
  return request.post<ApiResponse<PagedResult<ShopQueryDto>>>('/api/v1/shops/query', request);
};

export const exportShops = (request: ShopExportRequestDto) => {
  return request.post('/api/v1/shops/query/export', request, { responseType: 'blob' });
};
```

## 六、注意事項

### 6.1 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 必須使用快取機制（商店類型列表、城市列表、區域列表）
- 查詢條件過多時需要優化 SQL

### 6.2 資料驗證
- 查詢條件必須驗證
- 日期範圍必須驗證（起始日不能大於終止日）
- 分頁參數必須驗證

### 6.3 業務邏輯
- 查詢結果必須根據使用者權限過濾
- 匯出功能必須記錄操作日誌
- 大量資料匯出必須使用非同步處理
