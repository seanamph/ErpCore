# SYS3410-SYS3440 - 報表查詢作業系列 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: SYS3410-SYS3440 系列
- **功能名稱**: 報表查詢作業系列
- **功能描述**: 提供商店與會員相關報表的查詢、列印、匯出功能，包含會員統計報表、商店銷售報表、促銷活動效果報表等
- **參考舊程式**: 
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3410_*.ASP` (報表查詢相關功能)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3420_*.ASP` (報表查詢相關功能)
  - `WEB/IMS_CORE/ASP/SYS3000/SYS3440_*.ASP` (報表查詢相關功能)

### 1.2 業務需求
- 提供會員統計報表查詢
- 提供商店銷售報表查詢
- 提供促銷活動效果報表查詢
- 支援報表列印功能
- 支援報表匯出功能（Excel、PDF）
- 支援多條件篩選查詢
- 支援日期範圍查詢

## 二、資料庫設計 (Schema)

### 2.1 主要資料表

本功能主要使用現有的資料表進行報表查詢，不新增資料表。主要查詢的資料表包括：
- `Members` - 會員資料表
- `Shops` - 商店資料表
- `Promotions` - 促銷活動資料表
- `PromotionUsage` - 促銷活動使用記錄表
- `Orders` - 訂單資料表
- `OrderDetails` - 訂單明細資料表

### 2.2 報表查詢視圖

#### 2.2.1 `v_MemberStatistics` - 會員統計視圖
```sql
CREATE VIEW [dbo].[v_MemberStatistics] AS
SELECT 
    m.MemberId,
    m.MemberName,
    m.MemberLevel,
    m.JoinDate,
    m.Status,
    COUNT(DISTINCT o.OrderId) AS OrderCount,
    SUM(o.TotalAmount) AS TotalAmount,
    SUM(o.DiscountAmount) AS TotalDiscount,
    MAX(o.OrderDate) AS LastOrderDate
FROM [dbo].[Members] m
LEFT JOIN [dbo].[Orders] o ON m.MemberId = o.MemberId
GROUP BY m.MemberId, m.MemberName, m.MemberLevel, m.JoinDate, m.Status;
```

#### 2.2.2 `v_ShopSalesStatistics` - 商店銷售統計視圖
```sql
CREATE VIEW [dbo].[v_ShopSalesStatistics] AS
SELECT 
    s.ShopId,
    s.ShopName,
    o.OrderDate,
    COUNT(DISTINCT o.OrderId) AS OrderCount,
    SUM(o.TotalAmount) AS TotalAmount,
    SUM(o.DiscountAmount) AS TotalDiscount,
    COUNT(DISTINCT o.MemberId) AS MemberCount
FROM [dbo].[Shops] s
LEFT JOIN [dbo].[Orders] o ON s.ShopId = o.ShopId
GROUP BY s.ShopId, s.ShopName, o.OrderDate;
```

#### 2.2.3 `v_PromotionEffectStatistics` - 促銷活動效果統計視圖
```sql
CREATE VIEW [dbo].[v_PromotionEffectStatistics] AS
SELECT 
    p.PromoId,
    p.PromoName,
    p.PromoType,
    p.StartDate,
    p.EndDate,
    COUNT(DISTINCT pu.MemberId) AS MemberCount,
    COUNT(pu.TKey) AS UsageCount,
    SUM(pu.DiscountAmount) AS TotalDiscountAmount,
    AVG(pu.DiscountAmount) AS AvgDiscountAmount
FROM [dbo].[Promotions] p
LEFT JOIN [dbo].[PromotionUsage] pu ON p.PromoId = pu.PromoId
GROUP BY p.PromoId, p.PromoName, p.PromoType, p.StartDate, p.EndDate;
```

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢會員統計報表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/reports/member-statistics`
- **說明**: 查詢會員統計報表資料
- **請求參數**:
  ```json
  {
    "startDate": "2024-01-01",
    "endDate": "2024-12-31",
    "memberLevel": "",
    "status": "",
    "pageIndex": 1,
    "pageSize": 20
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "items": [
        {
          "memberId": "M001",
          "memberName": "張三",
          "memberLevel": "GOLD",
          "joinDate": "2024-01-01",
          "status": "A",
          "orderCount": 10,
          "totalAmount": 50000.00,
          "totalDiscount": 5000.00,
          "lastOrderDate": "2024-12-31"
        }
      ],
      "totalCount": 100,
      "pageIndex": 1,
      "pageSize": 20,
      "totalPages": 5
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.2 查詢商店銷售報表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/reports/shop-sales`
- **說明**: 查詢商店銷售報表資料
- **請求參數**:
  ```json
  {
    "startDate": "2024-01-01",
    "endDate": "2024-12-31",
    "shopId": "",
    "pageIndex": 1,
    "pageSize": 20
  }
  ```
- **回應格式**: 類似會員統計報表格式

#### 3.1.3 查詢促銷活動效果報表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/reports/promotion-effect`
- **說明**: 查詢促銷活動效果報表資料
- **請求參數**:
  ```json
  {
    "startDate": "2024-01-01",
    "endDate": "2024-12-31",
    "promoId": "",
    "promoType": "",
    "pageIndex": 1,
    "pageSize": 20
  }
  ```
- **回應格式**: 類似會員統計報表格式

#### 3.1.4 匯出報表（Excel）
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/reports/export-excel`
- **說明**: 匯出報表為Excel格式
- **請求格式**:
  ```json
  {
    "reportType": "MEMBER_STATISTICS", // MEMBER_STATISTICS, SHOP_SALES, PROMOTION_EFFECT
    "startDate": "2024-01-01",
    "endDate": "2024-12-31",
    "filters": {}
  }
  ```
- **回應格式**: 返回Excel檔案下載連結

#### 3.1.5 匯出報表（PDF）
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/reports/export-pdf`
- **說明**: 匯出報表為PDF格式
- **請求格式**: 同Excel匯出
- **回應格式**: 返回PDF檔案下載連結

#### 3.1.6 列印報表
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/reports/print`
- **說明**: 列印報表
- **請求格式**: 同Excel匯出
- **回應格式**: 返回列印資料

## 六、注意事項

### 6.1 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 必須使用快取機制
