- 管理促銷活動基本資料資訊

## 二、資料庫設計 (Schema)

### 2.1 主要資料表: `Promotions`

```sql
CREATE TABLE [dbo].[Promotions] (
    [TKey] BIGINT NOT NULL PRIMARY KEY IDENTITY(1,1),
    [PromoId] NVARCHAR(50) NOT NULL, -- 促銷活動編號
    [PromoName] NVARCHAR(200) NOT NULL, -- 促銷活動名稱
    [PromoNameEn] NVARCHAR(200) NULL, -- 促銷活動英文名稱
    [PromoType] NVARCHAR(50) NOT NULL, -- 促銷類型 (DISCOUNT:折扣, FULL_AMOUNT:滿額送, BUY_ONE_GET_ONE:買一送一, POINTS:積分, COUPON:優惠券)
    [StartDate] DATETIME2 NOT NULL, -- 活動開始日期
    [EndDate] DATETIME2 NOT NULL, -- 活動結束日期
    [StartTime] TIME NULL, -- 活動開始時間
    [EndTime] TIME NULL, -- 活動結束時間
    [DiscountType] NVARCHAR(50) NULL, -- 折扣類型 (PERCENTAGE:百分比, AMOUNT:金額)
    [DiscountValue] DECIMAL(18, 4) NULL, -- 折扣值
    [MinPurchaseAmount] DECIMAL(18, 4) NULL, -- 最低消費金額
    [MaxDiscountAmount] DECIMAL(18, 4) NULL, -- 最高折扣金額
    [MemberLevel] NVARCHAR(50) NULL, -- 適用會員等級（NULL表示全部）
    [ShopId] NVARCHAR(50) NULL, -- 適用商店（NULL表示全部）
    [ProductCategory] NVARCHAR(50) NULL, -- 適用商品分類（NULL表示全部）
    [ProductId] NVARCHAR(50) NULL, -- 適用商品（NULL表示全部）
    [MaxUsageCount] INT NULL, -- 最大使用次數（NULL表示無限制）
    [MaxUsagePerMember] INT NULL, -- 每會員最大使用次數（NULL表示無限制）
    [Status] NVARCHAR(10) NOT NULL DEFAULT 'A', -- 狀態 (A:啟用, I:停用, E:已結束)
    [Description] NVARCHAR(1000) NULL, -- 活動說明
    [Terms] NVARCHAR(2000) NULL, -- 活動條款
    [Notes] NVARCHAR(1000) NULL, -- 備註
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [UpdatedBy] NVARCHAR(50) NULL,
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [PK_Promotions] PRIMARY KEY CLUSTERED ([TKey] ASC),
    CONSTRAINT [UQ_Promotions_PromoId] UNIQUE ([PromoId])
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_Promotions_PromoId] ON [dbo].[Promotions] ([PromoId]);
CREATE NONCLUSTERED INDEX [IX_Promotions_PromoType] ON [dbo].[Promotions] ([PromoType]);
CREATE NONCLUSTERED INDEX [IX_Promotions_StartDate] ON [dbo].[Promotions] ([StartDate]);
CREATE NONCLUSTERED INDEX [IX_Promotions_EndDate] ON [dbo].[Promotions] ([EndDate]);
CREATE NONCLUSTERED INDEX [IX_Promotions_Status] ON [dbo].[Promotions] ([Status]);
CREATE NONCLUSTERED INDEX [IX_Promotions_ShopId] ON [dbo].[Promotions] ([ShopId]);
```

#### 3.1.2 查詢單筆促銷活動
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/promotions/{promoId}`
- **說明**: 根據促銷活動編號查詢單筆促銷活動資料
- **路徑參數**:
  - `promoId`: 促銷活動編號
- **回應格式**: 同查詢列表單筆資料格式，並包含適用商品列表

#### 3.1.4 修改促銷活動
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/promotions/{promoId}`
- **說明**: 修改促銷活動資料
- **路徑參數**:
  - `promoId`: 促銷活動編號
- **請求格式**: 同新增，但 `promoId` 不可修改
- **回應格式**: 同新增

#### 3.1.5 刪除促銷活動
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/promotions/{promoId}`
- **說明**: 刪除促銷活動資料（軟刪除或硬刪除）
- **路徑參數**:
  - `promoId`: 促銷活動編號
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "刪除成功",
    "data": null,
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.6 啟用/停用促銷活動
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/promotions/{promoId}/status`
- **說明**: 啟用或停用促銷活動
- **請求格式**:
  ```json
  {
    "status": "A" // A:啟用, I:停用, E:已結束
  }
  ```

#### 3.1.7 查詢促銷活動使用統計
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/promotions/{promoId}/statistics`
- **說明**: 查詢促銷活動使用統計資料
- **回應格式**:

## 六、注意事項

### 6.1 安全性
- 必須驗證使用者權限
- 敏感資料必須加密傳輸
- 促銷活動規則必須驗證

### 6.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 必須使用快取機制

### 6.3 資料驗證
- 促銷活動編號必須唯一
- 活動期間必須驗證（開始日期不能晚於結束日期）
- 折扣值必須在允許範圍內
- 狀態值必須在允許範圍內

### 6.4 業務邏輯
- 刪除促銷活動前必須檢查是否有使用記錄
- 停用促銷活動時必須檢查是否有進行中的訂單
- 促銷活動期間變更必須驗證
- 促銷活動規則必須驗證
