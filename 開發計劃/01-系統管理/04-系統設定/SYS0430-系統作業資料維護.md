# SYS0430 - 系統作業資料維護 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: SYS0430
- **功能名稱**: 系統作業資料維護
- **功能描述**: 提供系統作業資料的新增、修改、刪除、查詢功能，包含作業代碼、作業名稱、子系統項目代碼、網頁位址、作業型態、排序序號等資訊管理
- **參考舊程式**: 
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0430_FB.ASP` (瀏覽)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0430_FI.ASP` (新增)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0430_FU.ASP` (修改)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0430_FD.ASP` (刪除)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0430_FQ.ASP` (查詢)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0430_PR.ASP` (報表)

### 1.2 業務需求
- 管理系統作業資訊
- 支援作業的新增、修改、刪除、查詢
- 記錄作業的建立與變更資訊
- 支援作業型態設定
- 支援排序序號設定
- 與子系統、按鈕權限系統整合

## 二、資料庫設計 (Schema)

### 2.1 主要資料表: `Programs` (對應舊系統 `MNG_PROG`)

```sql
CREATE TABLE [dbo].[Programs] (
    [TKey] BIGINT NOT NULL PRIMARY KEY IDENTITY(1,1), -- 主鍵 (T_KEY)
    [ProgramId] NVARCHAR(50) NOT NULL, -- 作業代碼 (PROG_ID)
    [ProgramName] NVARCHAR(100) NOT NULL, -- 作業名稱 (PROG_NAME)
    [SeqNo] INT NULL, -- 排序序號 (SEQ_NO)
    [MenuId] NVARCHAR(50) NOT NULL, -- 子系統項目代碼 (MENU_ID)
    [ProgramUrl] NVARCHAR(500) NULL, -- 網頁位址 (URL)
    [ProgramType] NVARCHAR(20) NULL, -- 作業型態 (PROG_TYPE)
    [MaintainUserId] NVARCHAR(50) NULL, -- 維護者代碼 (MT_USER_ID)
    [Status] NVARCHAR(10) NOT NULL DEFAULT '1', -- 狀態 (STATUS)
    [CreatedBy] NVARCHAR(50) NULL, -- 建立者 (BUSER)
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(), -- 建立時間 (BTIME)
    [UpdatedBy] NVARCHAR(50) NULL, -- 更新者 (CUSER)
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(), -- 更新時間 (CTIME)
    [CreatedPriority] INT NULL, -- 建立者等級 (CPRIORITY)
    [CreatedGroup] NVARCHAR(50) NULL, -- 建立者群組 (CGROUP)
    CONSTRAINT [PK_Programs] PRIMARY KEY CLUSTERED ([TKey] ASC),
    CONSTRAINT [UQ_Programs_ProgramId] UNIQUE ([ProgramId]),
    CONSTRAINT [FK_Programs_Menus] FOREIGN KEY ([MenuId]) REFERENCES [dbo].[Menus] ([MenuId])
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_Programs_ProgramId] ON [dbo].[Programs] ([ProgramId]);
CREATE NONCLUSTERED INDEX [IX_Programs_MenuId] ON [dbo].[Programs] ([MenuId]);
CREATE NONCLUSTERED INDEX [IX_Programs_SeqNo] ON [dbo].[Programs] ([SeqNo]);
CREATE NONCLUSTERED INDEX [IX_Programs_ProgramType] ON [dbo].[Programs] ([ProgramType]);
```

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢作業列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/programs`
- **說明**: 查詢作業列表，支援分頁、排序、篩選
- **請求參數**:
  ```json
  {
    "pageIndex": 1,
    "pageSize": 20,
    "sortField": "ProgramId",
    "sortOrder": "ASC",
    "filters": {
      "programId": "",
      "programName": "",
      "menuId": "",
      "programType": ""
    }
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "items": [
        {
          "tKey": 1,
          "programId": "SYS0110",
          "programName": "使用者基本資料維護",
          "seqNo": 1,
          "menuId": "MENU001",
          "menuName": "使用者管理",
          "programUrl": "/system/users",
          "programType": "1",
          "programTypeName": "維護作業",
          "maintainUserId": "U001",
          "maintainUserName": "張三",
          "status": "1",
          "createdAt": "2024-01-01T00:00:00",
          "updatedAt": "2024-01-01T00:00:00"
        }
      ],
      "totalCount": 100,
      "pageIndex": 1,
      "pageSize": 20,
      "totalPages": 5
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.2 查詢單筆作業
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/programs/{programId}`
- **說明**: 根據作業代碼查詢單筆作業資料
- **路徑參數**:
  - `programId`: 作業代碼
- **回應格式**: 同查詢列表單筆資料

#### 3.1.3 新增作業
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/programs`
- **說明**: 新增作業資料
- **請求格式**:
  ```json
  {
    "programId": "SYS0110",
    "programName": "使用者基本資料維護",
    "seqNo": 1,
    "menuId": "MENU001",
    "programUrl": "/system/users",
    "programType": "1"
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "新增成功",
    "data": {
      "programId": "SYS0110"
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.4 修改作業
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/programs/{programId}`
- **說明**: 修改作業資料
- **路徑參數**:
  - `programId`: 作業代碼
- **請求格式**: 同新增，但 `programId` 不可修改
- **回應格式**: 同新增

#### 3.1.5 刪除作業
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/programs/{programId}`
- **說明**: 刪除作業資料（軟刪除或硬刪除）
- **路徑參數**:
  - `programId`: 作業代碼
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "刪除成功",
    "data": null,
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.6 批次刪除作業
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/programs/batch`
- **說明**: 批次刪除多筆作業
- **請求格式**:
  ```json
  {
    "programIds": ["SYS0110", "SYS0120", "SYS0130"]
  }
  ```

import request from '@/utils/request';

export interface ProgramDto {
  tKey: number;
  programId: string;
  programName: string;
  seqNo?: number;
  menuId: string;
  menuName?: string;
  programUrl?: string;
  programType?: string;
  programTypeName?: string;
  maintainUserId?: string;
  maintainUserName?: string;
  status: string;
  createdBy?: string;
  createdAt: string;
  updatedBy?: string;
  updatedAt: string;
}

export interface ProgramQueryDto {
  pageIndex: number;
  pageSize: number;
  sortField?: string;
  sortOrder?: 'ASC' | 'DESC';
  filters?: {
    programId?: string;
    programName?: string;
    menuId?: string;
    programType?: string;
  };
}

export interface CreateProgramDto {
  programId: string;
  programName: string;
  seqNo: number;
  menuId: string;
  programUrl: string;
  programType?: string;
}

export interface UpdateProgramDto extends Omit<CreateProgramDto, 'programId'> {}

// API 函數
export const getProgramList = (query: ProgramQueryDto) => {
  return request.get<ApiResponse<PagedResult<ProgramDto>>>('/api/v1/programs', { params: query });
};

export const getProgramById = (programId: string) => {
  return request.get<ApiResponse<ProgramDto>>(`/api/v1/programs/${programId}`);
};

export const createProgram = (data: CreateProgramDto) => {
  return request.post<ApiResponse<string>>('/api/v1/programs', data);
};

export const updateProgram = (programId: string, data: UpdateProgramDto) => {
  return request.put<ApiResponse>(`/api/v1/programs/${programId}`, data);
};

export const deleteProgram = (programId: string) => {
  return request.delete<ApiResponse>(`/api/v1/programs/${programId}`);
};

export const batchDeletePrograms = (programIds: string[]) => {
  return request.delete<ApiResponse>('/api/v1/programs/batch', { data: { programIds } });
};

## 七、注意事項

### 7.1 安全性
- 必須實作權限檢查
- 敏感資料必須加密傳輸 (HTTPS)

### 7.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引

### 7.3 資料驗證
- 作業代碼必須唯一
- 必填欄位必須驗證
- 子系統項目代碼必須存在
- 網頁位址格式驗證

### 7.4 業務邏輯
- 刪除作業前必須檢查是否有按鈕
- 維護者代碼自動設定為目前使用者
- 作業型態需從參數表讀取
