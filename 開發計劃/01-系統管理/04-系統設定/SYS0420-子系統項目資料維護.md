# SYS0420 - 子系統項目資料維護 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: SYS0420
- **功能名稱**: 子系統項目資料維護
- **功能描述**: 提供子系統項目資料的新增、修改、刪除、查詢功能，包含子系統代碼、子系統名稱、主系統代碼、上層子系統代碼、排序序號等資訊管理
- **參考舊程式**: 
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0420_FB.ASP` (瀏覽)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0420_FI.ASP` (新增)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0420_FU.ASP` (修改)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0420_FD.ASP` (刪除)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0420_FQ.ASP` (查詢)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0420_PR.ASP` (報表)

### 1.2 業務需求
- 管理系統子系統項目資訊
- 支援子系統的新增、修改、刪除、查詢
- 記錄子系統的建立與變更資訊
- 支援階層式子系統結構（上層子系統）
- 支援排序序號設定
- 與主系統、作業、按鈕權限系統整合

## 二、資料庫設計 (Schema)

### 2.1 主要資料表: `Menus` (對應舊系統 `MNG_MENU`)

```sql
CREATE TABLE [dbo].[Menus] (
    [TKey] BIGINT NOT NULL PRIMARY KEY IDENTITY(1,1), -- 主鍵 (T_KEY)
    [MenuId] NVARCHAR(50) NOT NULL, -- 子系統項目代碼 (MENU_ID)
    [MenuName] NVARCHAR(100) NOT NULL, -- 子系統項目名稱 (MENU_NAME)
    [SeqNo] INT NULL, -- 排序序號 (SEQ_NO)
    [SystemId] NVARCHAR(50) NOT NULL, -- 主系統代碼 (SYS_ID)
    [ParentMenuId] NVARCHAR(50) NULL, -- 上層子系統代碼 (P_MENU_ID)
    [Status] NVARCHAR(10) NOT NULL DEFAULT '1', -- 狀態 (STATUS)
    [CreatedBy] NVARCHAR(50) NULL, -- 建立者 (BUSER)
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(), -- 建立時間 (BTIME)
    [UpdatedBy] NVARCHAR(50) NULL, -- 更新者 (CUSER)
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(), -- 更新時間 (CTIME)
    [CreatedPriority] INT NULL, -- 建立者等級 (CPRIORITY)
    [CreatedGroup] NVARCHAR(50) NULL, -- 建立者群組 (CGROUP)
    CONSTRAINT [PK_Menus] PRIMARY KEY CLUSTERED ([TKey] ASC),
    CONSTRAINT [UQ_Menus_MenuId] UNIQUE ([MenuId]),
    CONSTRAINT [FK_Menus_Systems] FOREIGN KEY ([SystemId]) REFERENCES [dbo].[Systems] ([SystemId]),
    CONSTRAINT [FK_Menus_ParentMenu] FOREIGN KEY ([ParentMenuId]) REFERENCES [dbo].[Menus] ([MenuId])
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_Menus_MenuId] ON [dbo].[Menus] ([MenuId]);
CREATE NONCLUSTERED INDEX [IX_Menus_SystemId] ON [dbo].[Menus] ([SystemId]);
CREATE NONCLUSTERED INDEX [IX_Menus_ParentMenuId] ON [dbo].[Menus] ([ParentMenuId]);
CREATE NONCLUSTERED INDEX [IX_Menus_SeqNo] ON [dbo].[Menus] ([SeqNo]);
```

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢子系統列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/menus`
- **說明**: 查詢子系統列表，支援分頁、排序、篩選
- **請求參數**:
  ```json
  {
    "pageIndex": 1,
    "pageSize": 20,
    "sortField": "MenuId",
    "sortOrder": "ASC",
    "filters": {
      "menuId": "",
      "menuName": "",
      "systemId": "",
      "parentMenuId": ""
    }
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "items": [
        {
          "tKey": 1,
          "menuId": "MENU001",
          "menuName": "使用者管理",
          "seqNo": 1,
          "systemId": "SYS0000",
          "systemName": "系統管理",
          "parentMenuId": "0",
          "parentMenuName": "",
          "status": "1",
          "createdAt": "2024-01-01T00:00:00",
          "updatedAt": "2024-01-01T00:00:00"
        }
      ],
      "totalCount": 100,
      "pageIndex": 1,
      "pageSize": 20,
      "totalPages": 5
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.2 查詢單筆子系統
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/menus/{menuId}`
- **說明**: 根據子系統代碼查詢單筆子系統資料
- **路徑參數**:
  - `menuId`: 子系統代碼
- **回應格式**: 同查詢列表單筆資料

#### 3.1.3 新增子系統
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/menus`
- **說明**: 新增子系統資料
- **請求格式**:
  ```json
  {
    "menuId": "MENU001",
    "menuName": "使用者管理",
    "seqNo": 1,
    "systemId": "SYS0000",
    "parentMenuId": "0"
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "新增成功",
    "data": {
      "menuId": "MENU001"
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.4 修改子系統
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/menus/{menuId}`
- **說明**: 修改子系統資料
- **路徑參數**:
  - `menuId`: 子系統代碼
- **請求格式**: 同新增，但 `menuId` 不可修改
- **回應格式**: 同新增

#### 3.1.5 刪除子系統
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/menus/{menuId}`
- **說明**: 刪除子系統資料（軟刪除或硬刪除）
- **路徑參數**:
  - `menuId`: 子系統代碼
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "刪除成功",
    "data": null,
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.6 批次刪除子系統
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/menus/batch`
- **說明**: 批次刪除多筆子系統
- **請求格式**:
  ```json
  {
    "menuIds": ["MENU001", "MENU002", "MENU003"]
  }
  ```

### 4.3 API 呼叫 (`menu.api.ts`)
```typescript
import request from '@/utils/request';

export interface MenuDto {
  tKey: number;
  menuId: string;
  menuName: string;
  seqNo?: number;
  systemId: string;
  systemName?: string;
  parentMenuId?: string;
  parentMenuName?: string;
  status: string;
  createdBy?: string;
  createdAt: string;
  updatedBy?: string;
  updatedAt: string;
}

export interface MenuQueryDto {
  pageIndex: number;
  pageSize: number;
  sortField?: string;
  sortOrder?: 'ASC' | 'DESC';
  filters?: {
    menuId?: string;
    menuName?: string;
    systemId?: string;
    parentMenuId?: string;
  };
}

export interface CreateMenuDto {
  menuId: string;
  menuName: string;
  seqNo: number;
  systemId: string;
  parentMenuId: string;
}

export interface UpdateMenuDto extends Omit<CreateMenuDto, 'menuId'> {}

// API 函數
export const getMenuList = (query: MenuQueryDto) => {
  return request.get<ApiResponse<PagedResult<MenuDto>>>('/api/v1/menus', { params: query });
};

export const getMenuById = (menuId: string) => {
  return request.get<ApiResponse<MenuDto>>(`/api/v1/menus/${menuId}`);
};

export const createMenu = (data: CreateMenuDto) => {
  return request.post<ApiResponse<string>>('/api/v1/menus', data);
};

export const updateMenu = (menuId: string, data: UpdateMenuDto) => {
  return request.put<ApiResponse>(`/api/v1/menus/${menuId}`, data);
};

export const deleteMenu = (menuId: string) => {
  return request.delete<ApiResponse>(`/api/v1/menus/${menuId}`);
};

export const batchDeleteMenus = (menuIds: string[]) => {
  return request.delete<ApiResponse>('/api/v1/menus/batch', { data: { menuIds } });
};
```

## 七、注意事項

### 7.1 安全性
- 必須實作權限檢查
- 敏感資料必須加密傳輸 (HTTPS)

### 7.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 階層式查詢需考慮效能

### 7.3 資料驗證
- 子系統代碼必須唯一
- 必填欄位必須驗證
- 主系統代碼必須存在
- 上層子系統代碼必須存在（若不等於 '0'）
- 上層子系統代碼不能等於自己（避免循環引用）

### 7.4 業務邏輯
- 刪除子系統前必須檢查是否有下層子系統
- 刪除子系統前必須檢查是否有作業
- 上層子系統代碼若為 '0'，表示為根節點
- 支援階層式結構查詢
