# SYS0210 - 角色基本資料維護 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: SYS0210
- **功能名稱**: 角色基本資料維護
- **功能描述**: 提供角色基本資料的新增、修改、刪除、查詢功能，包含角色代碼、角色名稱、角色敘述等資訊管理
- **參考舊程式**: 
  - `IMS3/HANSHIN/RSL_CLASS/IMS3_SYS/MNG_ROLE.cs`
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0210_FI.ASP` (新增)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0210_FU.ASP` (修改)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0210_FD.ASP` (刪除)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0210_FQ.ASP` (查詢)
  - `WEB/IMS_CORE/SYS0000/SYS0210_PR.aspx` (報表)

### 1.2 業務需求
- 管理系統角色資訊
- 支援角色的新增、修改、刪除、查詢
- 記錄角色的建立與變更資訊
- 支援角色複製功能
- 與使用者權限系統整合

## 二、資料庫設計 (Schema)

### 2.1 主要資料表: `Roles`

```sql
CREATE TABLE [dbo].[Roles] (
    [RoleId] NVARCHAR(50) NOT NULL PRIMARY KEY,
    [RoleName] NVARCHAR(100) NOT NULL,
    [RoleNote] NVARCHAR(500) NULL,
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [UpdatedBy] NVARCHAR(50) NULL,
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [CreatedPriority] INT NULL,
    [CreatedGroup] NVARCHAR(50) NULL,
    CONSTRAINT [PK_Roles] PRIMARY KEY CLUSTERED ([RoleId] ASC)
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_Roles_RoleName] ON [dbo].[Roles] ([RoleName]);
```

### 2.2 相關資料表

#### 2.2.1 `UserRoles` - 使用者角色對應
```sql
CREATE TABLE [dbo].[UserRoles] (
    [UserId] NVARCHAR(50) NOT NULL,
    [RoleId] NVARCHAR(50) NOT NULL,
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [PK_UserRoles] PRIMARY KEY CLUSTERED ([UserId], [RoleId]),
    CONSTRAINT [FK_UserRoles_Users] FOREIGN KEY ([UserId]) REFERENCES [dbo].[Users] ([UserId]) ON DELETE CASCADE,
    CONSTRAINT [FK_UserRoles_Roles] FOREIGN KEY ([RoleId]) REFERENCES [dbo].[Roles] ([RoleId]) ON DELETE CASCADE
);
```

#### 2.2.2 `RolePermissions` - 角色權限對應
```sql
CREATE TABLE [dbo].[RolePermissions] (
    [RoleId] NVARCHAR(50) NOT NULL,
    [SystemId] NVARCHAR(50) NOT NULL,
    [ProgramId] NVARCHAR(50) NOT NULL,
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [PK_RolePermissions] PRIMARY KEY CLUSTERED ([RoleId], [SystemId], [ProgramId]),
    CONSTRAINT [FK_RolePermissions_Roles] FOREIGN KEY ([RoleId]) REFERENCES [dbo].[Roles] ([RoleId]) ON DELETE CASCADE
);
```

### 2.3 資料字典

| 欄位名稱 | 資料類型 | 長度 | 允許NULL | 預設值 | 說明 | 備註 |

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢角色列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/roles`
- **說明**: 查詢角色列表，支援分頁、排序、篩選
- **請求參數**:
  ```json
  {
    "pageIndex": 1,
    "pageSize": 20,
    "sortField": "RoleId",
    "sortOrder": "ASC",
    "filters": {
      "roleId": "",
      "roleName": ""
    }
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "items": [
        {
          "roleId": "R001",
          "roleName": "系統管理員",
          "roleNote": "系統管理員角色",
          "createdAt": "2024-01-01T00:00:00",
          "updatedAt": "2024-01-01T00:00:00"
        }
      ],
      "totalCount": 100,
      "pageIndex": 1,
      "pageSize": 20,
      "totalPages": 5
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.2 查詢單筆角色
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/roles/{roleId}`
- **說明**: 根據角色代碼查詢單筆角色資料
- **路徑參數**:
  - `roleId`: 角色代碼
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "roleId": "R001",
      "roleName": "系統管理員",
      "roleNote": "系統管理員角色",
      "createdBy": "U001",
      "createdAt": "2024-01-01T00:00:00",
      "updatedBy": "U001",
      "updatedAt": "2024-01-01T00:00:00"
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.3 新增角色
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/roles`
- **說明**: 新增角色資料
- **請求格式**:
  ```json
  {
    "roleId": "R001",
    "roleName": "系統管理員",
    "roleNote": "系統管理員角色"
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "新增成功",
    "data": {
      "roleId": "R001"
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.4 修改角色
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/roles/{roleId}`
- **說明**: 修改角色資料
- **路徑參數**:
  - `roleId`: 角色代碼
- **請求格式**: 同新增，但 `roleId` 不可修改
- **回應格式**: 同新增

#### 3.1.5 刪除角色
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/roles/{roleId}`
- **說明**: 刪除角色資料（需檢查是否有使用者使用此角色）
- **路徑參數**:
  - `roleId`: 角色代碼
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "刪除成功",
    "data": null,
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.6 複製角色
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/roles/{roleId}/copy`
- **說明**: 複製現有角色及其權限設定
- **請求格式**:
  ```json
  {
    "newRoleId": "R002",
    "newRoleName": "新角色名稱"
  }
  ```

### 4.3 API 呼叫 (`role.api.ts`)
```typescript
import request from '@/utils/request';

export interface RoleDto {
  roleId: string;
  roleName: string;
  roleNote?: string;
  createdBy?: string;
  createdAt?: string;
  updatedBy?: string;
  updatedAt?: string;
}

export interface RoleQueryDto {
  pageIndex: number;
  pageSize: number;
  sortField?: string;
  sortOrder?: 'ASC' | 'DESC';
  filters?: {
    roleId?: string;
    roleName?: string;
  };
}

export interface CreateRoleDto {
  roleId: string;
  roleName: string;
  roleNote?: string;
}

export interface UpdateRoleDto extends Omit<CreateRoleDto, 'roleId'> {}

export interface CopyRoleDto {
  newRoleId: string;
  newRoleName: string;
}

// API 函數
export const getRoleList = (query: RoleQueryDto) => {
  return request.get<ApiResponse<PagedResult<RoleDto>>>('/api/v1/roles', { params: query });
};

export const getRoleById = (roleId: string) => {
  return request.get<ApiResponse<RoleDto>>(`/api/v1/roles/${roleId}`);
};

export const createRole = (data: CreateRoleDto) => {
  return request.post<ApiResponse<string>>('/api/v1/roles', data);
};

export const updateRole = (roleId: string, data: UpdateRoleDto) => {
  return request.put<ApiResponse>(`/api/v1/roles/${roleId}`, data);
};

export const deleteRole = (roleId: string) => {
  return request.delete<ApiResponse>(`/api/v1/roles/${roleId}`);
};

export const copyRole = (roleId: string, data: CopyRoleDto) => {
  return request.post<ApiResponse<string>>(`/api/v1/roles/${roleId}/copy`, data);
};
```

## 六、注意事項

### 6.1 安全性
- 必須實作權限檢查
- 刪除角色前必須檢查是否有使用者使用此角色
- 敏感資料必須加密傳輸 (HTTPS)

### 6.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 必須使用快取機制

### 6.3 資料驗證
- 角色代碼必須唯一
- 必填欄位必須驗證
- 角色代碼長度限制 (最多50字元)
- 角色名稱長度限制 (最多100字元)

### 6.4 業務邏輯
- 刪除角色前必須檢查是否有使用者使用此角色
- 複製角色時必須同時複製權限設定
- 角色變更必須記錄變更資訊
