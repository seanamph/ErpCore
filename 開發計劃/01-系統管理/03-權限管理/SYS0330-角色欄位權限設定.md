# SYS0330 - 角色欄位權限設定 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: SYS0330
- **功能名稱**: 角色欄位權限設定
- **功能描述**: 提供角色欄位權限設定的查詢、設定功能，用於控制角色對資料庫欄位的存取權限
- **參考舊程式**: 
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0330_FQ.ASP` (查詢)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0330_DB_LIST.asp` (資料庫列表)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0330_TABLE_LIST.asp` (表格列表)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0330_FIELD_LIST.asp` (欄位列表)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0330_ROLE_LIST.asp` (角色列表)
  - `WEB/IMS_CORE/ASP/SYS0000/SYS0330_SET_FIELD_AC.asp` (設定欄位權限)

### 1.2 業務需求
- 管理角色對資料庫欄位的存取權限
- 支援依資料庫、表格、欄位設定權限
- 支援角色欄位權限的查詢
- 支援欄位權限的設定與修改
- 支援多資料庫環境

### 4.3 API 呼叫 (`roleFieldPermission.api.ts`)
```typescript
import request from '@/utils/request';

export interface RoleFieldPermissionDto {
  id: string;
  roleId: string;
  roleName?: string;
  dbName: string;
  tableName: string;
  fieldName: string;
  permissionType: string;
  createdBy?: string;
  createdAt?: string;
}

export interface DatabaseDto {
  dbName: string;
  dbDescription: string;
}

export interface TableDto {
  tableName: string;
  tableDescription: string;
}

export interface FieldDto {
  fieldName: string;
  fieldType: string;
  fieldLength?: number;
  fieldDescription: string;
}

// API 函數
export const getDatabases = () => {
  return request.get<ApiResponse<List<DatabaseDto>>>('/api/v1/role-field-permissions/databases');
};

export const getTables = (dbName: string) => {
  return request.get<ApiResponse<List<TableDto>>>('/api/v1/role-field-permissions/tables', { params: { dbName } });
};

export const getFields = (dbName: string, tableName: string) => {
  return request.get<ApiResponse<List<FieldDto>>>('/api/v1/role-field-permissions/fields', { params: { dbName, tableName } });
};

export const getPermissions = (query: RoleFieldPermissionQueryDto) => {
  return request.get<ApiResponse<List<RoleFieldPermissionDto>>>('/api/v1/role-field-permissions', { params: query });
};

export const createPermission = (data: CreateRoleFieldPermissionDto) => {
  return request.post<ApiResponse<string>>('/api/v1/role-field-permissions', data);
};

export const updatePermission = (id: string, data: UpdateRoleFieldPermissionDto) => {
  return request.put<ApiResponse>(`/api/v1/role-field-permissions/${id}`, data);
};

export const deletePermission = (id: string) => {
  return request.delete<ApiResponse>(`/api/v1/role-field-permissions/${id}`);
};

export const batchSetPermissions = (data: BatchSetRoleFieldPermissionDto) => {
  return request.post<ApiResponse>('/api/v1/role-field-permissions/batch', data);
};
```

## 六、注意事項

### 6.1 安全性
- 必須實作權限檢查
- 敏感欄位必須有適當的權限控制
- 必須驗證角色權限

### 6.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 必須使用快取機制

### 6.3 資料驗證
- 角色代碼必須存在
- 資料庫、表格、欄位必須存在
- 權限類型必須在允許範圍內

### 6.4 業務邏輯
- 同一角色對同一欄位只能有一個權限設定
- 刪除權限前必須檢查是否有相關資料
- 權限變更必須記錄
