# SYS0110 - 使用者基本資料維護 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: SYS0110
- **功能名稱**: 使用者基本資料維護
- **功能描述**: 提供使用者基本資料的新增、修改、刪除、查詢功能，包含使用者帳號、密碼、組織、權限等資訊管理
- **參考舊程式**: 
  - `IMS3/HANSHIN/IMS3/SYS0000/SYS0110.ascx`
  - `WEB/IMS_CORE/SYS0000/SYS0110_FI.aspx` (新增)
  - `WEB/IMS_CORE/SYS0000/SYS0110_FU.aspx` (修改)
  - `WEB/IMS_CORE/SYS0000/SYS0110_FD.aspx` (刪除)
  - `WEB/IMS_CORE/SYS0000/SYS0110_FQ.aspx` (查詢)
  - `WEB/IMS_CORE/SYS0000/SYS0110_PR.aspx` (報表)

### 1.2 業務需求
- 管理系統使用者帳號資訊
- 支援使用者帳號的啟用/停用
- 記錄使用者登入資訊
- 支援密碼管理
- 支援使用者權限設定
- 支援多組織架構

### 2.1 主要資料表: `Users`

```sql
CREATE TABLE [dbo].[Users] (
    [UserId] NVARCHAR(50) NOT NULL PRIMARY KEY,
    [UserName] NVARCHAR(100) NOT NULL,
    [UserPassword] NVARCHAR(255) NULL,
    [Title] NVARCHAR(50) NULL,
    [OrgId] NVARCHAR(50) NULL,
    [StartDate] DATETIME2 NULL,
    [EndDate] DATETIME2 NULL,
    [LastLoginDate] DATETIME2 NULL,
    [LastLoginIp] NVARCHAR(50) NULL,
    [Status] NVARCHAR(10) NOT NULL DEFAULT 'A', -- A:啟用, I:停用, L:鎖定
    [UserType] NVARCHAR(20) NULL,
    [Notes] NVARCHAR(500) NULL,
    [UserPriority] INT NULL DEFAULT 0,
    [ShopId] NVARCHAR(50) NULL,
    [LoginCount] INT NULL DEFAULT 0,
    [ChangePwdDate] DATETIME2 NULL,
    [FloorId] NVARCHAR(50) NULL,
    [AreaId] NVARCHAR(50) NULL,
    [BtypeId] NVARCHAR(50) NULL,
    [StoreId] NVARCHAR(50) NULL,
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [UpdatedBy] NVARCHAR(50) NULL,
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [CreatedPriority] INT NULL,
    [CreatedGroup] NVARCHAR(50) NULL,
    CONSTRAINT [PK_Users] PRIMARY KEY CLUSTERED ([UserId] ASC)
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_Users_OrgId] ON [dbo].[Users] ([OrgId]);
```

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢使用者列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/users`
- **說明**: 查詢使用者列表，支援分頁、排序、篩選
- **請求參數**:
  ```json
  {
    "pageIndex": 1,
    "pageSize": 20,
    "sortField": "UserId",
    "sortOrder": "ASC",
    "filters": {
      "userId": "",
      "userName": "",
      "orgId": "",
      "status": "",
      "userType": ""
    }
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "items": [
        {
          "userId": "U001",
          "userName": "張三",
          "title": "經理",
          "orgId": "ORG001",
          "status": "A",
          "userType": "INTERNAL",
          "lastLoginDate": "2024-01-01T10:00:00",
          "loginCount": 100
        }
      ],
      "totalCount": 100,
      "pageIndex": 1,
      "pageSize": 20,
      "totalPages": 5
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.2 查詢單筆使用者
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/users/{userId}`
- **說明**: 根據使用者編號查詢單筆使用者資料
- **路徑參數**:
  - `userId`: 使用者編號
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "userId": "U001",
      "userName": "張三",
      "title": "經理",
      "orgId": "ORG001",
      "startDate": "2024-01-01",
      "endDate": null,
      "status": "A",
      "userType": "INTERNAL",
      "notes": "備註",
      "userPriority": 1,
      "shopId": "SHOP001",
      "loginCount": 100,
      "lastLoginDate": "2024-01-01T10:00:00",
      "lastLoginIp": "192.168.1.1"
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.3 新增使用者
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/users`
- **說明**: 新增使用者資料
- **請求格式**:
  ```json
  {
    "userId": "U001",
    "userName": "張三",
    "userPassword": "Password123!",
    "title": "經理",
    "orgId": "ORG001",
    "startDate": "2024-01-01",
    "endDate": null,
    "status": "A",
    "userType": "INTERNAL",
    "notes": "備註",
    "userPriority": 1,
    "shopId": "SHOP001"
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "新增成功",
    "data": {
      "userId": "U001"
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.4 修改使用者
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/users/{userId}`
- **說明**: 修改使用者資料
- **路徑參數**:
  - `userId`: 使用者編號
- **請求格式**: 同新增，但 `userId` 不可修改
- **回應格式**: 同新增

#### 3.1.5 刪除使用者
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/users/{userId}`
- **說明**: 刪除使用者資料（軟刪除或硬刪除）
- **路徑參數**:
  - `userId`: 使用者編號
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "刪除成功",
    "data": null,
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.6 批次刪除使用者
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/users/batch`
- **說明**: 批次刪除多筆使用者
- **請求格式**:
  ```json
  {
    "userIds": ["U001", "U002", "U003"]
  }
  ```

#### 3.1.7 修改密碼
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/users/{userId}/password`
- **說明**: 修改使用者密碼
- **請求格式**:
  ```json
  {
    "oldPassword": "OldPassword123!",
    "newPassword": "NewPassword123!"
  }
  ```

#### 3.1.8 重設密碼
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/users/{userId}/reset-password`
- **說明**: 管理員重設使用者密碼
- **請求格式**:
  ```json
  {
    "newPassword": "NewPassword123!"
  }
  ```

#### 3.1.9 啟用/停用使用者
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/users/{userId}/status`
- **說明**: 啟用或停用使用者帳號
- **請求格式**:
  ```json
  {
    "status": "A" // A:啟用, I:停用, L:鎖定
  }
  ```

### 4.3 API 呼叫 (`user.api.ts`)
```typescript
import request from '@/utils/request';

export interface UserDto {
  userId: string;
  userName: string;
  title?: string;
  orgId?: string;
  startDate?: string;
  endDate?: string;
  status: string;
  userType?: string;
  notes?: string;
  userPriority?: number;
  shopId?: string;
  loginCount?: number;
  lastLoginDate?: string;
  lastLoginIp?: string;
}

export interface UserQueryDto {
  pageIndex: number;
  pageSize: number;
  sortField?: string;
  sortOrder?: 'ASC' | 'DESC';
  filters?: {
    userId?: string;
    userName?: string;
    orgId?: string;
    status?: string;
    userType?: string;
  };
}

export interface CreateUserDto {
  userId: string;
  userName: string;
  userPassword: string;
  title?: string;
  orgId?: string;
  startDate?: string;
  endDate?: string;
  status: string;
  userType?: string;
  notes?: string;
  userPriority?: number;
  shopId?: string;
}

export interface UpdateUserDto extends Omit<CreateUserDto, 'userId' | 'userPassword'> {}

// API 函數
export const getUserList = (query: UserQueryDto) => {
  return request.get<ApiResponse<PagedResult<UserDto>>>('/api/v1/users', { params: query });
};

export const getUserById = (userId: string) => {
  return request.get<ApiResponse<UserDto>>(`/api/v1/users/${userId}`);
};

export const createUser = (data: CreateUserDto) => {
  return request.post<ApiResponse<string>>('/api/v1/users', data);
};

export const updateUser = (userId: string, data: UpdateUserDto) => {
```

## 六、注意事項

### 6.1 安全性
- 密碼必須加密儲存 (BCrypt)
- 密碼必須符合複雜度要求
- 敏感資料必須加密傳輸 (HTTPS)
- 必須實作權限檢查

### 6.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 必須使用快取機制

### 6.3 資料驗證
- 使用者編號必須唯一
- 必填欄位必須驗證
- 日期範圍必須驗證
- 狀態值必須在允許範圍內

### 6.4 業務邏輯
- 刪除使用者前必須檢查是否有相關資料
- 停用使用者時必須檢查是否有進行中的業務
- 密碼變更必須記錄變更日期
- 登入資訊必須記錄
