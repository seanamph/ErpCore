# SYS0111 - 使用者基本資料維護(業種儲位) 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: SYS0111
- **功能名稱**: 使用者基本資料維護(業種儲位)
- **功能描述**: 提供使用者基本資料的新增、修改、刪除、查詢功能，特別針對業種大分類、中分類、小分類以及儲位1、儲位2、儲位3的設定管理
- **參考舊程式**: 
  - `IMS3/HANSHIN/IMS3/SYS0000/SYS0111.ascx`
  - `IMS3/HANSHIN/IMS3/SYS0000/SYS0111.ascx.cs`

### 1.2 業務需求
- 管理系統使用者帳號資訊
- 支援使用者業種權限設定（大分類、中分類、小分類）
- 支援使用者儲位權限設定（儲位1、儲位2、儲位3）
- 支援7X承租分店代號設定
- 支援使用者帳號的啟用/停用
- 記錄使用者登入資訊
- 支援密碼管理
- 支援使用者權限設定
- 支援多組織架構

### 2.2 相關資料表

#### 2.2.1 `UserBusinessTypes` - 使用者業種權限
```sql
CREATE TABLE [dbo].[UserBusinessTypes] (
    [Id] BIGINT NOT NULL PRIMARY KEY IDENTITY(1,1),
    [UserId] NVARCHAR(50) NOT NULL,
    [BtypeMId] NVARCHAR(50) NULL, -- 業種大分類
    [BtypeId] NVARCHAR(50) NULL, -- 業種中分類
    [PtypeId] NVARCHAR(50) NULL, -- 業種小分類
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [FK_UserBusinessTypes_Users] FOREIGN KEY ([UserId]) REFERENCES [dbo].[Users] ([UserId]) ON DELETE CASCADE
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_UserBusinessTypes_UserId] ON [dbo].[UserBusinessTypes] ([UserId]);
CREATE NONCLUSTERED INDEX [IX_UserBusinessTypes_BtypeMId] ON [dbo].[UserBusinessTypes] ([BtypeMId]);
CREATE NONCLUSTERED INDEX [IX_UserBusinessTypes_BtypeId] ON [dbo].[UserBusinessTypes] ([BtypeId]);
```

#### 2.2.2 `UserWarehouseAreas` - 使用者儲位權限
```sql
CREATE TABLE [dbo].[UserWarehouseAreas] (
    [Id] BIGINT NOT NULL PRIMARY KEY IDENTITY(1,1),
    [UserId] NVARCHAR(50) NOT NULL,
    [WareaId1] NVARCHAR(50) NULL, -- 儲位1
    [WareaId2] NVARCHAR(50) NULL, -- 儲位2
    [WareaId3] NVARCHAR(50) NULL, -- 儲位3
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [FK_UserWarehouseAreas_Users] FOREIGN KEY ([UserId]) REFERENCES [dbo].[Users] ([UserId]) ON DELETE CASCADE
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_UserWarehouseAreas_UserId] ON [dbo].[UserWarehouseAreas] ([UserId]);
CREATE NONCLUSTERED INDEX [IX_UserWarehouseAreas_WareaId1] ON [dbo].[UserWarehouseAreas] ([WareaId1]);
```

#### 2.2.3 `UserStores` - 使用者7X承租分店權限
```sql
CREATE TABLE [dbo].[UserStores] (
```

### 3.1 API 端點列表

#### 3.1.1 查詢使用者列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/users`
- **說明**: 查詢使用者列表，支援分頁、排序、篩選
- **請求參數**: 參考 `SYS0110-使用者基本資料維護.md`
- **回應格式**: 參考 `SYS0110-使用者基本資料維護.md`

- **路徑**: `/api/v1/users/{userId}/detail`

  - `userId`: 使用者編號
- **回應格式**:
  ```json

          "btypeId": "BT001001",
          "btypeName": "生鮮",
          "ptypeId": "BT001001001",
          "ptypeName": "蔬菜"
        }
      ],
      "warehouseAreas": [
        {
          "wareaId1": "WA001",
          "wareaName1": "儲位A",
          "wareaId2": "WA002",
          "wareaName2": "儲位B",
          "wareaId3": "WA003",
  ```

- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/users/{userId}`
- **說明**: 修改使用者資料，包含業種、儲位、7X承租分店權限設定
- **路徑參數**:

- **請求格式**: 同新增

- **路徑**: `/api/v1/business-types/major`

### 4.3 API 呼叫 (`user.api.ts`)
參考 `SYS0110-使用者基本資料維護.md`，並新增以下函數：

```typescript
export const getUserDetail = (userId: string) => {
  return request.get<ApiResponse<UserDetailDto>>(`/api/v1/users/${userId}/detail`);
};

export const getBusinessTypeMajors = () => {
  return request.get<ApiResponse<List<BusinessTypeMajorDto>>>('/api/v1/business-types/major');
};

export const getBusinessTypeMiddles = (btypeMId: string) => {
  return request.get<ApiResponse<List<BusinessTypeMiddleDto>>>(`/api/v1/business-types/middle?btypeMId=${btypeMId}`);
};

export const getBusinessTypeMinors = (btypeId: string) => {
  return request.get<ApiResponse<List<BusinessTypeMinorDto>>>(`/api/v1/business-types/minor?btypeId=${btypeId}`);
};

export const getWarehouseAreas = (level?: number, parentId?: string) => {
  return request.get<ApiResponse<List<WarehouseAreaDto>>>('/api/v1/warehouse-areas', {
    params: { level, parentId }
  });
```

## 六、注意事項

### 6.1 業務邏輯
- 業種設定為階層式（大分類→中分類→小分類），選擇上層時需載入下層選項
- 儲位設定為階層式（儲位1→儲位2→儲位3），選擇上層時需載入下層選項
- 業種大分類可選擇「全部業種」選項（值為"xx"）
- 儲位1可選擇「全部儲位」選項（值為"xx"）
- 7X承租分店可多選
- 刪除使用者時需同步刪除相關的業種、儲位、分店權限

### 6.2 資料驗證
- 業種設定必須符合階層關係
- 儲位設定必須符合階層關係
- 7X承租分店代號必須存在

### 6.3 效能
- 業種、儲位下拉選單需使用快取機制
- 大量資料查詢必須使用分頁
