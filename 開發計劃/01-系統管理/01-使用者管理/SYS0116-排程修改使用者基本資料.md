# SYS0116 - 排程修改使用者基本資料 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: SYS0116
- **功能名稱**: 排程修改使用者基本資料
- **功能描述**: 提供使用者基本資料的排程修改功能，可設定未來某個時間點自動執行使用者資料的修改，包含密碼重設等操作
- **參考舊程式**: 
  - `IMS3/HANSHIN/IMS3/SYS0000/SYS0116.ascx`
  - `IMS3/HANSHIN/IMS3/SYS0000/SYS0116.ascx.cs`

### 1.2 業務需求
- 支援排程修改使用者基本資料
- 支援排程時間設定（日期、時、分、秒）
- 支援排程密碼重設
- 支援排程狀態管理（待執行、執行中、已完成、已取消）
- 支援排程執行記錄查詢
- 支援排程取消功能
- 記錄排程操作日誌

## 二、資料庫設計 (Schema)

### 2.1 主要資料表: `UserSchedules`

```sql
CREATE TABLE [dbo].[UserSchedules] (
    [ScheduleId] UNIQUEIDENTIFIER NOT NULL PRIMARY KEY DEFAULT NEWID(),
    [UserId] NVARCHAR(50) NOT NULL,
    [ScheduleDate] DATETIME2 NOT NULL, -- 排程執行時間
    [ScheduleType] NVARCHAR(20) NOT NULL, -- 排程類型: PASSWORD_RESET, USER_UPDATE, STATUS_CHANGE
    [Status] NVARCHAR(20) NOT NULL DEFAULT 'PENDING', -- 狀態: PENDING, EXECUTING, COMPLETED, CANCELLED, FAILED
    [ScheduleData] NVARCHAR(MAX) NULL, -- 排程資料（JSON格式）
    [ExecuteResult] NVARCHAR(MAX) NULL, -- 執行結果
    [ErrorMessage] NVARCHAR(500) NULL, -- 錯誤訊息
    [ExecutedAt] DATETIME2 NULL, -- 實際執行時間
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [UpdatedBy] NVARCHAR(50) NULL,
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [FK_UserSchedules_Users] FOREIGN KEY ([UserId]) REFERENCES [dbo].[Users] ([UserId]) ON DELETE CASCADE
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_UserSchedules_UserId] ON [dbo].[UserSchedules] ([UserId]);
CREATE NONCLUSTERED INDEX [IX_UserSchedules_ScheduleDate] ON [dbo].[UserSchedules] ([ScheduleDate]);
CREATE NONCLUSTERED INDEX [IX_UserSchedules_Status] ON [dbo].[UserSchedules] ([Status]);
CREATE NONCLUSTERED INDEX [IX_UserSchedules_ScheduleType] ON [dbo].[UserSchedules] ([ScheduleType]);
```

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢排程列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/user-schedules`
- **說明**: 查詢使用者排程列表，支援分頁、排序、篩選
- **請求參數**:
  ```json
  {
    "pageIndex": 1,
    "pageSize": 20,
    "sortField": "ScheduleDate",
    "sortOrder": "ASC",
    "filters": {
      "userId": "",
      "status": "",
      "scheduleType": "",
      "scheduleDateFrom": "",
      "scheduleDateTo": ""
    }
  }
  ```

#### 3.1.2 查詢單筆排程
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/user-schedules/{scheduleId}`
- **說明**: 根據排程編號查詢單筆排程資料

#### 3.1.3 新增排程
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/user-schedules`
- **說明**: 新增使用者排程
- **請求格式**:
  ```json
  {
    "userId": "U001",
    "scheduleDate": "2024-01-15T10:00:00",
    "scheduleType": "PASSWORD_RESET",
    "scheduleData": {
      "newPassword": "NewPassword123!",
      "autoGenerate": false
    }
  }
  ```

#### 3.1.4 修改排程
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/user-schedules/{scheduleId}`
- **說明**: 修改排程（僅限待執行狀態）
- **路徑參數**:
  - `scheduleId`: 排程編號

#### 3.1.5 取消排程
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/user-schedules/{scheduleId}/cancel`
- **說明**: 取消排程（僅限待執行狀態）

#### 3.1.6 執行排程（系統內部）
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/user-schedules/{scheduleId}/execute`
- **說明**: 執行排程（由排程服務呼叫）

## 六、注意事項

### 6.1 排程執行
- 需實作背景服務定期檢查待執行的排程
- 排程執行時間需考慮時區問題
- 排程執行失敗需記錄錯誤訊息
- 排程執行成功需更新狀態和執行時間

### 6.2 業務邏輯
- 僅待執行狀態的排程可修改或取消
- 排程執行時間必須在未來
- 排程資料需以JSON格式儲存
- 需記錄排程操作日誌

### 6.3 效能
- 大量排程查詢必須使用分頁
- 背景服務需控制執行頻率，避免過度負載
