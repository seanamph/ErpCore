# CFG0420 - 子系統項目資料維護 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: CFG0420
- **功能名稱**: 子系統項目資料維護
- **功能描述**: 提供配置模組子系統項目資料的新增、修改、刪除、查詢功能，包含子系統代碼、子系統名稱、主系統代碼、上層子系統代碼、排序序號等資訊管理
- **參考舊程式**: 
  - `IMS3/HANSHIN/IMS3/CFG0000/CFG0420_FI1.aspx` (新增)
  - `IMS3/HANSHIN/IMS3/CFG0000/CFG0420_FU1.aspx` (修改)
  - `IMS3/HANSHIN/IMS3/CFG0000/CFG0420_FD.aspx` (刪除)
  - `IMS3/HANSHIN/IMS3/CFG0000/CFG0420_PR.aspx` (報表)

### 1.2 業務需求
- 管理配置模組子系統項目資訊
- 支援子系統的新增、修改、刪除、查詢
- 記錄子系統的建立與變更資訊
- 支援階層式子系統結構（上層子系統）
- 支援排序序號設定
- 與主系統、作業、按鈕權限系統整合

## 二、資料庫設計 (Schema)

### 2.1 主要資料表: `ConfigSubSystems` (對應舊系統配置模組子系統)

```sql
CREATE TABLE [dbo].[ConfigSubSystems] (
    [TKey] BIGINT NOT NULL PRIMARY KEY IDENTITY(1,1), -- 主鍵 (T_KEY)
    [SubSystemId] NVARCHAR(50) NOT NULL, -- 子系統項目代碼 (SUB_SYS_ID)
    [SubSystemName] NVARCHAR(100) NOT NULL, -- 子系統項目名稱 (SUB_SYS_NAME)
    [SeqNo] INT NULL, -- 排序序號 (SEQ_NO)
    [SystemId] NVARCHAR(50) NOT NULL, -- 主系統代碼 (SYS_ID)
    [ParentSubSystemId] NVARCHAR(50) NULL, -- 上層子系統代碼 (P_SUB_SYS_ID)
    [Status] NVARCHAR(10) NOT NULL DEFAULT 'A', -- 狀態 (STATUS, A:啟用, I:停用)
    [Notes] NVARCHAR(500) NULL, -- 備註 (NOTES)
    [CreatedBy] NVARCHAR(50) NULL, -- 建立者 (BUSER)
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(), -- 建立時間 (BTIME)
    [UpdatedBy] NVARCHAR(50) NULL, -- 更新者 (CUSER)
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(), -- 更新時間 (CTIME)
    [CreatedPriority] INT NULL, -- 建立者等級 (CPRIORITY)
    [CreatedGroup] NVARCHAR(50) NULL, -- 建立者群組 (CGROUP)
    CONSTRAINT [PK_ConfigSubSystems] PRIMARY KEY CLUSTERED ([TKey] ASC),
    CONSTRAINT [UQ_ConfigSubSystems_SubSystemId] UNIQUE ([SubSystemId]),
    CONSTRAINT [FK_ConfigSubSystems_ConfigSystems] FOREIGN KEY ([SystemId]) REFERENCES [dbo].[ConfigSystems] ([SystemId]) ON DELETE CASCADE,
    CONSTRAINT [FK_ConfigSubSystems_ParentSubSystem] FOREIGN KEY ([ParentSubSystemId]) REFERENCES [dbo].[ConfigSubSystems] ([SubSystemId])
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_ConfigSubSystems_SubSystemId] ON [dbo].[ConfigSubSystems] ([SubSystemId]);
CREATE NONCLUSTERED INDEX [IX_ConfigSubSystems_SystemId] ON [dbo].[ConfigSubSystems] ([SystemId]);
CREATE NONCLUSTERED INDEX [IX_ConfigSubSystems_ParentSubSystemId] ON [dbo].[ConfigSubSystems] ([ParentSubSystemId]);
CREATE NONCLUSTERED INDEX [IX_ConfigSubSystems_SeqNo] ON [dbo].[ConfigSubSystems] ([SeqNo]);
```

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢子系統列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/config/subsystems`
- **說明**: 查詢配置模組子系統列表，支援分頁、排序、篩選
- **請求參數**:
  ```json
  {
    "pageIndex": 1,
    "pageSize": 20,
    "sortField": "SubSystemId",
    "sortOrder": "ASC",
    "filters": {
      "subSystemId": "",
      "subSystemName": "",
      "systemId": "",
      "parentSubSystemId": "",
      "status": ""
    }
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "查詢成功",
    "data": {
      "items": [
        {
          "tKey": 1,
          "subSystemId": "SUB001",
          "subSystemName": "使用者管理",
          "seqNo": 1,
          "systemId": "SYS0000",
          "systemName": "系統管理",
          "parentSubSystemId": "0",
          "parentSubSystemName": "",
          "status": "A",
          "notes": "備註",
          "createdAt": "2024-01-01T00:00:00",
          "updatedAt": "2024-01-01T00:00:00"
        }
      ],
      "totalCount": 100,
      "pageIndex": 1,
      "pageSize": 20,
      "totalPages": 5
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.2 查詢單筆子系統
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/config/subsystems/{subSystemId}`
- **說明**: 根據子系統代碼查詢單筆子系統資料
- **路徑參數**:
  - `subSystemId`: 子系統代碼
- **回應格式**: 同查詢列表單筆資料

#### 3.1.3 新增子系統
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/config/subsystems`
- **說明**: 新增子系統資料
- **請求格式**:
  ```json
  {
    "subSystemId": "SUB001",
    "subSystemName": "使用者管理",
    "seqNo": 1,
    "systemId": "SYS0000",
    "parentSubSystemId": "0",
    "status": "A",
    "notes": "備註"
  }
  ```
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "新增成功",
    "data": {
      "subSystemId": "SUB001"
    },
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.4 修改子系統
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/config/subsystems/{subSystemId}`
- **說明**: 修改子系統資料
- **路徑參數**:
  - `subSystemId`: 子系統代碼
- **請求格式**: 同新增，但 `subSystemId` 不可修改
- **回應格式**: 同新增

#### 3.1.5 刪除子系統
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/config/subsystems/{subSystemId}`
- **說明**: 刪除子系統資料（需檢查是否有下層子系統或作業）
- **路徑參數**:
  - `subSystemId`: 子系統代碼
- **回應格式**:
  ```json
  {
    "success": true,
    "code": 200,
    "message": "刪除成功",
    "data": null,
    "timestamp": "2024-01-01T00:00:00Z"
  }
  ```

#### 3.1.6 批次刪除子系統
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/config/subsystems/batch`
- **說明**: 批次刪除多筆子系統
- **請求格式**:
  ```json
  {
    "subSystemIds": ["SUB001", "SUB002", "SUB003"]
  }
  ```

### 3.5 Controller: `ConfigSubSystemsController.cs`
```csharp
namespace RSL.IMS3.Api.Controllers
{
    [ApiController]
    [Route("api/v1/config/subsystems")]
    [Authorize]
    public class ConfigSubSystemsController : ControllerBase
    {
        private readonly IConfigSubSystemService _configSubSystemService;
        
        public ConfigSubSystemsController(IConfigSubSystemService configSubSystemService)
        {
            _configSubSystemService = configSubSystemService;
        }
        
        [HttpGet]
        public async Task<ActionResult<ApiResponse<PagedResult<ConfigSubSystemDto>>>> GetConfigSubSystems([FromQuery] ConfigSubSystemQueryDto query)
        {
            var result = await _configSubSystemService.GetConfigSubSystemsAsync(query);
            return Ok(ApiResponse<PagedResult<ConfigSubSystemDto>>.Success(result));
        }
        
        [HttpGet("{subSystemId}")]
        public async Task<ActionResult<ApiResponse<ConfigSubSystemDto>>> GetConfigSubSystem(string subSystemId)
        {
            var result = await _configSubSystemService.GetConfigSubSystemByIdAsync(subSystemId);
            return Ok(ApiResponse<ConfigSubSystemDto>.Success(result));
        }
        
        [HttpPost]
        public async Task<ActionResult<ApiResponse<string>>> CreateConfigSubSystem([FromBody] CreateConfigSubSystemDto dto)
        {
            var subSystemId = await _configSubSystemService.CreateConfigSubSystemAsync(dto);
            return Ok(ApiResponse<string>.Success(subSystemId));
        }
        
        [HttpPut("{subSystemId}")]
        public async Task<ActionResult<ApiResponse>> UpdateConfigSubSystem(string subSystemId, [FromBody] UpdateConfigSubSystemDto dto)
        {
            await _configSubSystemService.UpdateConfigSubSystemAsync(subSystemId, dto);
            return Ok(ApiResponse.Success());
        }
        
        [HttpDelete("{subSystemId}")]
        public async Task<ActionResult<ApiResponse>> DeleteConfigSubSystem(string subSystemId)
        {
            await _configSubSystemService.DeleteConfigSubSystemAsync(subSystemId);
            return Ok(ApiResponse.Success());
        }
        
        [HttpDelete("batch")]
        public async Task<ActionResult<ApiResponse>> BatchDeleteConfigSubSystems([FromBody] BatchDeleteConfigSubSystemsRequestDto request)
        {
            await _configSubSystemService.BatchDeleteConfigSubSystemsAsync(request.SubSystemIds);
            return Ok(ApiResponse.Success());
        }
    }
}
```

## 六、注意事項

### 6.1 安全性
- 必須實作權限檢查
- 敏感資料必須加密傳輸 (HTTPS)

### 6.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 階層式查詢需考慮效能

### 6.3 資料驗證
- 子系統代碼必須唯一
- 必填欄位必須驗證
- 主系統代碼必須存在
- 上層子系統代碼必須存在（若不等於 '0'）
- 上層子系統代碼不能等於自己（避免循環引用）

### 6.4 業務邏輯
- 刪除子系統前必須檢查是否有下層子系統
- 刪除子系統前必須檢查是否有作業
- 上層子系統代碼若為 '0'，表示為根節點
- 支援階層式結構查詢
