# CUS3000 - 客戶定制模組系列 開發計劃

## 一、功能概述

### 1.1 功能說明
- **功能代碼**: CUS3000 系列
- **功能名稱**: 客戶定制模組系列
- **功能描述**: 提供客戶定制3000系列功能，包含會員管理、促銷活動、交易處理等客戶定制功能。此模組為特定客戶的定制化功能，包含SYS3130-SYS3160系列（會員管理）、SYS3210-SYS3299系列（會員相關功能）、SYS3310-SYS3399系列（促銷活動）、SYS3410-SYS3440系列、SYS3510-SYS3580系列（活動管理）等功能
- **參考舊程式**: 
  - `WEB/IMS_CORE/ASP/CUS3000/SYS3130_*.ASP` (會員管理)
  - `WEB/IMS_CORE/ASP/CUS3000/SYS3210_*.ASP` (會員相關功能)
  - `WEB/IMS_CORE/ASP/CUS3000/SYS3310_*.ASP` (促銷活動)
  - `WEB/IMS_CORE/ASP/CUS3000/SYS3410_*.ASP` (相關功能)
  - `WEB/IMS_CORE/ASP/CUS3000/SYS3510_*.ASP` (活動管理)

### 1.2 業務需求
- 管理會員基本資料（SYS3130-SYS3160系列）
- 支援會員卡片管理
- 支援會員照片上傳
- 支援會員查詢功能
- 管理促銷活動（SYS3310-SYS3399系列）
- 支援促銷活動管理、憑證管理、背景處理、查詢等功能
- 管理活動資料（SYS3510-SYS3580系列）
- 支援活動管理功能

## 二、資料庫設計 (Schema)

### 2.1 主要資料表: `Cus3000Members` (會員主檔)

```sql
CREATE TABLE [dbo].[Cus3000Members] (
    [TKey] BIGINT NOT NULL PRIMARY KEY IDENTITY(1,1),
    [MemberId] NVARCHAR(50) NOT NULL, -- 會員編號
    [MemberName] NVARCHAR(100) NOT NULL, -- 會員姓名
    [CardNo] NVARCHAR(50) NULL, -- 會員卡號
    [Phone] NVARCHAR(20) NULL, -- 電話
    [Email] NVARCHAR(100) NULL, -- 電子郵件
    [Address] NVARCHAR(500) NULL, -- 地址
    [BirthDate] DATETIME2 NULL, -- 出生日期
    [Gender] NVARCHAR(10) NULL, -- 性別 (M:男, F:女)
    [PhotoPath] NVARCHAR(500) NULL, -- 照片路徑
    [Status] NVARCHAR(10) NOT NULL DEFAULT 'A', -- 狀態 (A:啟用, I:停用)
    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [UpdatedBy] NVARCHAR(50) NULL,
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    CONSTRAINT [UQ_Cus3000Members_MemberId] UNIQUE ([MemberId]),
    CONSTRAINT [UQ_Cus3000Members_CardNo] UNIQUE ([CardNo])
);

-- 索引
CREATE NONCLUSTERED INDEX [IX_Cus3000Members_MemberId] ON [dbo].[Cus3000Members] ([MemberId]);
CREATE NONCLUSTERED INDEX [IX_Cus3000Members_CardNo] ON [dbo].[Cus3000Members] ([CardNo]);
CREATE NONCLUSTERED INDEX [IX_Cus3000Members_Status] ON [dbo].[Cus3000Members] ([Status]);
```

    [CreatedBy] NVARCHAR(50) NULL,
    [CreatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),
    [UpdatedBy] NVARCHAR(50) NULL,
    [UpdatedAt] DATETIME2 NOT NULL DEFAULT GETDATE(),

## 三、後端 API 設計

### 3.1 API 端點列表

#### 3.1.1 查詢會員列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/cus3000/members`
- **說明**: 查詢會員列表，支援分頁、排序、篩選
- **請求參數**: 標準查詢參數（pageIndex, pageSize, sortField, sortOrder, filters）
- **回應格式**: 標準列表回應格式

#### 3.1.2 查詢單筆會員
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/cus3000/members/{memberId}`
- **說明**: 查詢單筆會員資料

#### 3.1.3 新增會員
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/cus3000/members`
- **說明**: 新增會員資料

#### 3.1.4 修改會員
- **HTTP 方法**: `PUT`
- **路徑**: `/api/v1/cus3000/members/{memberId}`
- **說明**: 修改會員資料

#### 3.1.5 刪除會員
- **HTTP 方法**: `DELETE`
- **路徑**: `/api/v1/cus3000/members/{memberId}`
- **說明**: 刪除會員資料

#### 3.1.6 上傳會員照片
- **HTTP 方法**: `POST`
- **路徑**: `/api/v1/cus3000/members/{memberId}/photo`
- **說明**: 上傳會員照片

#### 3.1.7 查詢促銷活動列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/cus3000/promotions`
- **說明**: 查詢促銷活動列表

#### 3.1.8 查詢活動列表
- **HTTP 方法**: `GET`
- **路徑**: `/api/v1/cus3000/activities`
- **說明**: 查詢活動列表

## 六、注意事項

### 6.1 安全性
- 敏感資料必須加密傳輸 (HTTPS)
- 必須實作權限檢查
- 照片上傳必須驗證檔案類型

### 6.2 效能
- 大量資料查詢必須使用分頁
- 必須建立適當的索引
- 照片上傳必須限制檔案大小

### 6.3 資料驗證
- 會員編號必須唯一
- 會員卡號必須唯一
- 必填欄位必須驗證

### 6.4 業務邏輯
- 客戶定制功能必須符合客戶特定需求
- 促銷活動日期必須驗證
- 活動日期必須驗證

### 6.5 客戶定制需求
- 必須與客戶確認功能需求
- 必須符合客戶業務流程
- 必須支援客戶特定報表
